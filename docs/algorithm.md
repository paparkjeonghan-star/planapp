자동 채우기 알고리즘 설계

목표
- 주중 시간표의 `study` 슬롯을 기반으로 과목별 주간 목표 시간(또는 우선순위)을 분배하여 `StudySession`을 생성
- 다음 규칙을 지원: 우선순위 가중치, 최소 세션 길이, 동일 과목의 연속 배정 회피

주요 파라미터
- `minSessionMinutes` (기본 30): 하나 세션의 최소 길이(분). 슬롯과 과목 잔여 시간에 따라 유연하게 적용.
- `avoidConsecutive` (기본 true): 인접 슬롯에서 동일 과목 연속 배정을 우회하려 시도.

알고리즘 요약
1. 시간표의 모든 슬롯 길이(분)를 합산해 총 사용 가능 시간을 계산.
2. 과목별 목표시간 합(`totalTargetHours`)이 있으면 과목 `priority`를 가중치로 곱한 비율로 분배.
   - 목표시간이 없으면 `priority` 비율로 분배.
3. 슬롯을 시간순(요일, 시작시간)으로 순회하며 각 슬롯을 빈 공간부터 채움:
   - 잔여시간이 가장 많은 과목을 우선 선택하되, `avoidConsecutive`가 설정되어 있으면 직전 배정 과목과 다르게 선택하려 시도.
   - 선택된 과목에 대해 가능한 시간을 배정(최대: 슬롯 남은 시간, 과목 잔여시간).
   - 최소 세션 길이 규칙을 만족하도록 조정하며, 불가피한 잔여는 허용.
4. 생성된 `StudySession`을 DB에 저장하고, `Allocation`의 `hours`와 `sessions`를 업데이트.

확장 포인트
- 과목별 금지 시간대(예: 특정 요일/시간대 학습 불가) 반영
- 동일 과목의 연속 제한을 보다 엄격히(예: 연속 2세션 금지) 적용
- 학생 선호(예: 아침/저녁 선호)와 피로도 모델을 도입해 스케줄 최적화
- 제약 만족형(최적화) 접근: ILP/CP-SAT를 이용한 최적 배치

API
- `POST /api/plans/generate`에서 `options`에 위 파라미터를 전달하여 동작을 조정할 수 있습니다.
  예: `{ studentId, weekStart, options: { minSessionMinutes: 20, avoidConsecutive: false } }`
