데이터 모델 초안

엔티티

1. `User`
- `id` (UUID)
- `email` (string, unique)
- `name` (string)
- `role` (enum: admin, teacher, student)
- `createdAt` (timestamp)

2. `Student`
- `id` (UUID)
- `userId` (UUID, nullable)  // 학생 계정이 있는 경우
- `name` (string)
- `grade` (string)
- `notes` (text)

3. `Subject`
- `id` (UUID)
- `studentId` (UUID) or null for global subjects
- `name` (string)
- `weeklyTargetHours` (float)
- `priority` (int)
- `color` (string)

4. `Timetable`
- `id` (UUID)
- `studentId` (UUID)
- `weekStart` (date)  // ISO week start date
- `slots` (relation to `Slot`)

5. `Slot`
- `id` (UUID)
- `timetableId` (UUID)
- `dayOfWeek` (int 0-6)
- `startTime` (time)
- `endTime` (time)
- `type` (enum: study, other)
- `subjectId` (UUID, optional)  // 사전에 배정된 과목

6. `Plan`
- `id` (UUID)
- `studentId` (UUID)
- `weekStart` (date)
- `generatedAt` (timestamp)
- `autoGenerated` (boolean)
- `allocations` (relation to `Allocation`)

7. `Allocation` (과목별 배분)
- `id` (UUID)
- `planId` (UUID)
- `subjectId` (UUID)
- `hours` (float)
- `sessions` (int)

8. `StudySession`
- `id` (UUID)
- `planId` (UUID)
- `subjectId` (UUID)
- `date` (date)
- `startTime` (time)
- `durationMinutes` (int)
- `completed` (boolean)
- `notes` (text)

9. `Template`
- `id` (UUID)
- `name` (string)
- `structure` (JSON)  // 예: 주간 슬롯 템플릿

관계와 제약
- 한 `Student`는 여러 `Timetable`, `Plan`, `Subject`를 가짐
- `Plan`은 특정 주의 `Timetable`을 기반으로 생성

자동 배분 알고리즘 입력
- 주당 과목별 목표 시간(`weeklyTargetHours`)
- 시간표의 여유 슬롯(주중에 `type=study`인 슬롯)
- 우선순위(`priority`)와 학생 선호

출력
- 각 슬롯에 배분된 `StudySession` 목록
- 각 과목별 할당(`Allocation`) 합계

비고
- 시간대 및 로컬타임 처리를 명확히 할 것
- 반복 템플릿(매주 동일 시간표) 지원 권장
